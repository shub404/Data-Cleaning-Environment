# DataClean-RL: Data Cleaning Environment

DataClean-RL is a reinforcement learning environment for evaluating how agents perform multi-step data cleaning on structured datasets.

The environment simulates common real-world data issues such as missing values, duplicates, inconsistent formatting, and outliers. Agents interact with the environment by selecting cleaning actions over a fixed number of steps, with the goal of maximizing final data quality.

---

## Motivation

Most data cleaning systems rely on fixed pipelines or isolated transformations. In practice, cleaning requires sequential decisions:

- identifying which issue to fix first  
- choosing whether to modify or leave data unchanged  
- avoiding unnecessary or destructive operations  

This environment is designed to test whether an agent can make these decisions step by step under constraints.

---

## Environment

### Observation

At each step, the agent receives:

- a preview of the dataset (first 10 rows)  
- a list of detected issues  

Issue types include:

- duplicates  
- missing_values:<column>  
- format_inconsistency:<column>  
- outlier_detected:<column>  

This representation provides a compact view of dataset quality without exposing the full dataset.

---

### Actions

The agent can take one of the following actions:

- `remove_duplicates`  
- `fill_missing` (optionally for a specific column)  
- `normalize_text` (for a given column)  
- `llm_clean` (generic cleaning step)  
- `done` (terminate the episode)  

---

### Episode

- maximum steps: 10  
- the agent interacts sequentially with the dataset  
- the episode ends when `done` is selected or step limit is reached  

---

## Reward

The environment uses both intermediate rewards and a final evaluation.

### Step rewards

- removing duplicates: +0.2  
- filling missing values: +0.1  
- normalizing text: +0.1  
- llm_clean: +0.15  

### Final reward

At the end of the episode, a score is computed by comparing the cleaned dataset to a reference version.

The reference dataset is generated by:

- removing duplicates  
- filling missing values  
- normalizing text  

The final reward is a normalized score between 0 and 1 representing how close the result is to the reference.

---

## Tasks

The environment supports three difficulty levels.

| Difficulty | Description |
|-----------|------------|
| Easy | Missing values in a simple dataset |
| Medium | Duplicates and inconsistent text formatting |
| Hard | Combination of missing values, outliers, and mixed formats |

## Usage

### Install dependencies
uv sync

### Run environment server
uv run server

### Run planner
python inference_planner.py

### Run benchmark
python benchmark.py

## Notes

- the environment is deterministic for a given dataset sample  
- the reference dataset is generated programmatically, not manually labeled  
- the focus is on decision quality, not just transformation accuracy
